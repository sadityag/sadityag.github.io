What are Recamán Numbers?

The Recamán sequence, named after Colombian mathematician Bernardo Recamán Santos, is a fascinating integer sequence that creates beautiful spiral patterns when visualized. The sequence is defined by a simple rule that leads to surprisingly complex and unpredictable behavior.

Starting with R(0) = 0, each subsequent term is generated by the rule:

R(n) = R(n-1) - n if R(n-1) - n > 0 and R(n-1) - n is not already in the sequence

R(n) = R(n-1) + n otherwise

This deceptively simple rule creates a sequence that jumps back and forth along the number line, always trying to subtract first, but adding when subtraction would create a negative number or repeat a value already in the sequence.

Mathematical Properties

The Recamán sequence exhibits several intriguing mathematical properties:

Sequence Behavior

The first few terms of the sequence are: 0, 1, 3, 6, 2, 7, 13, 20, 12, 21, 11, 22, 10, 23, 9, 24, 8, 25...

Open Conjectures

Several fascinating questions about the Recamán sequence remain open:

  • Does every positive integer eventually appear in the sequence?
  • What is the density of integers that appear in the sequence?
  • Are there infinitely many integers that never appear?

Computational Challenges

Computing large terms of the Recamán sequence requires careful tracking of which numbers have already appeared, making it an interesting problem for algorithmic optimization and data structure design.

Visualization & Results

When plotted as semicircles connecting consecutive terms, the Recamán sequence creates a beautiful spiral pattern that has captivated mathematicians and artists alike. Each semicircle connects R(n-1) to R(n), creating an intricate web of curves.

Computational Implementation

The Mathematica notebook includes several key computational components:

Memoized Sequence Generation

The core algorithm uses memoization to efficiently compute the Recamán sequence. Here's the implementation:

Recaman[0] = 0;
Recaman[1] = 1;
Recaman[n_] := 
 Recaman[n] = 
  Recaman[n - 1] - 
   If[Not[MemberQ[Table[Recaman[i], {i, 0, n - 1}], 
       Recaman[n - 1] - n]] && (Recaman[n - 1] - n > 0), n, -n]

This approach stores previously computed values to avoid redundant calculations, though the algorithm still faces computational challenges due to the membership testing required at each step. The need to check if a value already exists in the sequence makes this an inherently expensive operation for large n.
